# Orflow

> Recurring billing for Nomba merchants. No rebuild required.

A managed subscription billing API built on Nomba's payment primitives. Orflow handles plan management, card tokenization, scheduled charges, proration, dunning, and outbound webhooks — so you don't have to build billing infrastructure from scratch.

Built by **Team Blueprint** for the **DevCareer x Nomba Hackathon 2026**.

---

## Stack

| Layer | Technology |
|-------|-----------|
| Bundler | Vite 8 (Rolldown) |
| UI | React 19, TypeScript 6 |
| Routing | TanStack Router (file-based) |
| Styling | Tailwind CSS v4, shadcn/ui (radix-nova) |
| Icons | @solar-icons/react |
| State | TanStack Query, zustand |
| HTTP | axios |
| Fonts | Manrope Variable (body/headings), Geist Mono Variable (code) |

### Solar-icons note

`@solar-icons/react` v1.1.1 has a massive barrel export that chokes Vite 8's Rolldown pre-bundler. This project uses direct-path imports through `src/lib/icons.ts` — a local barrel that imports each icon from its individual ESM file via the package's `exports` map (`@solar-icons/react/icons/{category}/{IconName}`). This bypasses the barrel completely and avoids the pre-bundling issue.

## API

The backend lives at **[github.com/Team-Blueprint/orflow-backend](https://github.com/Team-Blueprint/orflow-backend)**.

Orflow provides:

- `POST /plans` — Create a subscription plan
- `POST /checkouts` — Generate a Nomba checkout link (card tokenization)
- Webhook endpoints for payment lifecycle events
- Dashboard for managing plans, subscribers, and billing history
